---
title: Shared State
description: The .coral/ directory and knowledge sharing between agents.
---


The `.coral/` directory is the central hub for all shared state in a CORAL run. It's created by `coral prepare`; each agent sees `.coral/public/` through its runtime-native shared directory such as `.claude/`, `.codex/`, or `.opencode/`. The worktree root also has a `CORAL_SHARED` symlink to that runtime directory.

## Directory structure

```
.coral/
├── config.yaml              # Copy of the task configuration
├── public/                  # Shared across all agents
│   ├── attempts/            # Eval results (JSON files)
│   │   ├── <commit-hash>.json
│   │   └── ...
│   ├── knowledge/
│   │   ├── manuals/         # Stable CORAL operating docs
│   │   ├── external/        # Frozen papers, repos, docs, datasets, references
│   │   ├── briefs/          # Codex-prepared runnable agent seed bundles
│   │   └── practice/        # Eval-linked agent chains, notebooks, reflections
│   ├── skills/              # Reusable tools (directories)
│   │   ├── optimizer/
│   │   │   ├── SKILL.md
│   │   │   └── tool.py
│   │   └── ...
│   ├── logs/                # Agent session logs
│   ├── eval_count           # Global eval counter (integer)
│   ├── grader_daemon.pid    # PID of the grader daemon process
│   └── grader_daemon_liveness  # Daemon liveness timestamp (ISO 8601)
│
└── private/                 # Hidden from agents
    ├── grader_venv/         # Isolated venv running the grader entrypoint
    └── grader_checkouts/    # Temporary worktrees for grading (ephemeral)
```

## Attempts

Every time an agent runs `coral eval`, an attempt record is written to `.coral/public/attempts/`:

```json
{
  "commit_hash": "abc1234",
  "agent_id": "agent-1",
  "title": "Optimized inner loop with vectorization",
  "score": 0.85,
  "status": "improved",
  "parent_hash": "def5678",
  "timestamp": "2025-03-15T10:30:00+00:00",
  "feedback": "eval: Runtime reduced from 2.3s to 1.1s"
}
```

### Status values

| Status | Meaning |
|--------|---------|
| `pending` | Submitted by agent, waiting for grader daemon to score |
| `improved` | Score is better than this agent's previous best |
| `baseline` | Score equals the previous best |
| `regressed` | Score is worse than the previous best |
| `crashed` | Grader raised an exception |
| `timeout` | Grader exceeded the timeout |

## Knowledge

Agents do not browse shared knowledge by listing every file. They query indexes first, then read the specific item they need:

```bash
coral kb index manual
coral kb index external
coral kb index practice --by score
coral kb read <id>
```

External knowledge is treated as frozen input. Practice knowledge is mutable run output: eval-linked method summaries, score curves, notebooks, and reflections that point back to the exact commit via `coral show <hash> --diff`.

## Skills

Skills are reusable tools that agents package for other agents. Each skill is a directory with a `SKILL.md` descriptor:

```
skills/
└── profiler/
    ├── SKILL.md       # Description and usage instructions
    └── profile.py     # The actual tool
```

Browse skills:

```bash
coral skills                   # List all skills
coral skills --read profiler   # Show skill details
```

## Access control

The workspace guard hook enforces boundaries:

| Path | Agents can... |
|------|---------------|
| Own worktree | Read + Write |
| Sibling worktrees | Read only |
| Runtime shared dir / `CORAL_SHARED` | Read + Write only through public shared state and CLI contracts |
| `.coral/private/` | No access |

This ensures agents can collaborate through shared state while keeping grader internals hidden.
