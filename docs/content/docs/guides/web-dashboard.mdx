CORAL includes a web dashboard for real-time monitoring of agent runs.

## Launch

```bash
coral ui
```

Opens `http://127.0.0.1:8420` in your browser. Customize the port:

```bash
coral ui --port 9000
coral ui --no-open          # Don't auto-open browser
```

You can also launch the dashboard alongside a prepared run:

```bash
coral start -c results/<task>/latest/.coral/config.yaml run.ui=true
```

## Dashboard tabs

### Control

Start, stop, and configure a run:
- **Resource budget** — CPU, GPU, memory, and storage hints for eval jobs
- **Agents and eval** — backend, model, reasoning effort, eval script, and eval profile
- **Run** — total runtime, readiness gate, attempts, eval count, and token usage

### Overview

Monitor live progress:
- **Score trajectory** — Real-time plot of scores over eval sequence, with baseline lines when baselines are recorded
- **Agent states** — `work_loop`, `reflect_loop`, waiting/eval state, duration, token usage, and current eval progress

### Knowledge

Browse indexed run knowledge:
- **External knowledge** — papers, repos, docs, datasets, and web references
- **Practice knowledge** — per-agent eval sequence, score, eval report, method archive, and route summaries
- **Skills** — reusable tools available to agents

Low-level trace logs and eval logs are still written under `.coral/public/` for
debugging and feedback, but they are not a top-level dashboard tab.

## Backend

The dashboard uses:
- **Starlette** — Python ASGI backend
- **Server-Sent Events (SSE)** — Live updates without polling
- **React + Vite** — TypeScript frontend

The backend reads directly from the `.coral/` directory, so the dashboard always reflects the latest state.

## Targeting a specific run

By default, `coral ui` auto-detects the latest active run. Target a specific run:

```bash
coral ui --task my-task --run 2025-03-15_10-30-00
```
